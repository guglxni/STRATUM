# STRATUM diagrams

Architecture and contract-flow visuals for README, `docs/ARCHITECTURE.md`, and hackathon submission. Sources are version-controlled; **SVG** (docs/GitHub) and **PNG** (slides/Notion) are generated by one script.

## Prerequisites

```bash
# draw.io (already present if brew cask installed)
brew install --cask drawio

# Mermaid CLI via npx (no global install required)
npx -p @mermaid-js/mermaid-cli mmdc --version
```

## Render everything

```bash
./scripts/render-diagrams.sh
```

Outputs:

| Format | Directory | Use |
|--------|-----------|-----|
| SVG | `docs/diagrams/svg/` | GitHub README, dark/light themes |
| PNG | `docs/diagrams/png/` | Presentations, PDF, social |

The primary architecture and flow diagrams are authored as draw.io files and generated from
`scripts/drawio_gen.py` + `scripts/build_diagrams.py` (a small generator that keeps colors, fonts,
rounded nodes, and labeled orthogonal edges consistent). Regenerate the draw.io sources then render:

```bash
python3 scripts/build_diagrams.py    # rewrite the .drawio sources
./scripts/render-diagrams.sh         # render every .mmd + .drawio to SVG + PNG
```

### Manual single file (draw.io)

```bash
/Applications/draw.io.app/Contents/MacOS/draw.io \
  -x -f svg --no-sandbox --crop \
  -o docs/diagrams/svg/stratum-architecture.svg \
  docs/diagrams/drawio/stratum-architecture.drawio
```

### Manual single file (Mermaid)

```bash
npx -p @mermaid-js/mermaid-cli mmdc \
  -i docs/diagrams/mermaid/hook-lifecycle.mmd \
  -o docs/diagrams/png/hook-lifecycle.png \
  -b white --scale 2
```

## CI note

SVG/PNG artifacts are **committed** so `forge test` and GitHub Actions do not require Chrome or draw.io. Re-run the script after changing any `.mmd` or `.drawio` source.
